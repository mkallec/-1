# 宝塔服务器部署说明

## 项目概况

这是一个纯静态HTML网站（日韩影视网），包含：
- ~2000个HTML页面
- 150张JPG海报图片（根目录）
- 10个MP4视频文件（videos/ 目录）
- CSS/JS资源（assets/ 目录）

**无需PHP、Node.js、数据库**，直接用Nginx或Apache托管即可。

---

## 部署步骤

### 1. 上传文件到服务器

将整个项目文件夹上传到宝塔服务器的网站目录，例如：
```
/www/wwwroot/你的域名/
```

**注意：** videos/ 文件夹中的MP4文件是本地视频源，必须一起上传。

### 2. 宝塔面板配置

在宝塔面板中：

1. **添加站点**：进入"网站" → "添加站点"，填写你的域名
2. **网站目录**：指向你上传的项目根目录
3. **运行环境**：选择纯静态（不需要PHP）

### 3. Nginx配置（可选优化）

如果你的视频文件较大，建议在宝塔的"伪静态"或Nginx配置中添加以下内容，优化视频播放和缓存：

```nginx
# 视频文件缓存
location ~* \.(mp4|m3u8)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
    add_header Accept-Ranges bytes;
    mp4;
    mp4_buffer_size 4M;
    mp4_max_buffer_size 10M;
}

# 图片缓存
location ~* \.(jpg|jpeg|png|gif|webp)$ {
    expires 90d;
    add_header Cache-Control "public, immutable";
}

# CSS/JS 缓存
location ~* \.(css|js)$ {
    expires 30d;
    add_header Cache-Control "public, immutable";
}
```

### 4. 域名和SSL

- 在宝塔面板中给你的网站添加SSL证书（可以使用Let's Encrypt免费证书）
- 确保域名解析到你的服务器IP

## 关于视频播放

网站的视频播放有两种模式：

### 模式一：HLS流播放（Cloudflare Stream）
```html
data-play="https://customer-xxx.cloudflarestream.com/.../video.m3u8"
```
这是远程视频源，**与Netlify无关**，如果你的Cloudflare Stream还在付费，部署后仍可正常使用。

### 模式二：本地MP4文件
```html
<source src="videos/1.mp4" type="video/mp4">
```
videos/ 目录下有10个MP4文件作为本地视频源。如需添加更多视频，放到videos/目录即可。
